# נתוני דמו לחודש מלא

האפליקציה כוללת מערך נתונים סינתטי ודטרמיניסטי המדמה חודש עבודה של קבוצת כדורגל. כל מזהי הדמו מתחילים ב-`demo-month-`, והערות הפעילות מסומנות ב-`[DEMO חודש מלא]`.

## טעינה אוטומטית

כאשר `RPE_SEED_DEMO_DATA=true`:

- במצב Local, הנתונים נטענים אוטומטית אם אין state קיים ב-`localStorage`.
- במצב Supabase, הנתונים נטענים אוטומטית רק כאשר הקבוצה או רשימת השחקנים ריקות.

## טעינה או רענון ידני

1. פתחו `/coach/settings`.
2. לחצו `טעינה / רענון חודש דמו`.
3. אשרו את הודעת הבטיחות.

הפעולה מחליפה רק רשומות עם מזהי דמו ידועים. נתונים אמיתיים שכבר קיימים ב-state או ב-Supabase נשארים ללא שינוי.

ב-Supabase הפעולה מוחקת תחילה רק IDs עם קידומת `demo-month-`, ואז מבצעת upsert של מערך החודש החדש יחד עם כל הנתונים האמיתיים שכבר נטענו. מזהי מערך הדמו הישן מוסרים רק אם כל ה-state מזוהה בוודאות כמערך הדמו הישן הטהור; ב-state מעורב הם נשמרים כדי לא לסכן נתון אמיתי.

## כניסת שחקנים

לכל שחקן PIN דטרמיניסטי:

- השחקן הראשון: `1001`
- השחקן השני: `1002`
- וכן הלאה עד השחקן ה-22: `1022`

ה-PIN מוצג וניתן לעריכה במסך ההגדרות.

## טווח הנתונים

החודש מעוגן לתאריך שבו בוצעה הטעינה:

- ארבעה שבועות שהושלמו.
- ארבעה אימונים לפני כל משחק: `MD-4`, `MD-3`, `MD-2`, `MD-1`.
- ארבעה משחקים: ליגה, גביע, משחק אימון וליגה.
- `MD+1` אחרי כל משחק.
- משחק השבוע הרביעי מתקיים יום לפני תאריך העוגן.
- `MD+1` של השבוע הרביעי מתקיים בתאריך העוגן.
- חמישה אירועים עתידיים עם עומסי GPS/RPE מתוכננים.

## אימות

```powershell
node scripts/verify-demo-data.mjs
npm.cmd run check
```

הבדיקה מאמתת כמויות, עמדות, GPS מותאם עמדה, חוסרים, כאבים חוזרים, מגמת שינה, הידרציה, עומס GPS גבוה ו-RPE גבוה.

לא נדרשת מיגרציית מסד נתונים. צד ועוצמת כאב בדוחות RPE נשמרים בתוך metadata תואם לאחור בשדה `comments`.
